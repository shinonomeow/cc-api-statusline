# Documentation Index

This repo has two documentation layers:

1. **Current implementation docs** (authoritative for new work)
2. **Legacy spec docs** (deprecated reference only)

## Authoritative docs (read first)

- `docs/current-implementation.md` - current runtime model and architecture as implemented in `src/`
- `docs/ccstatusline-contract-reference.md` - source-backed host runtime contract
- `docs/perf-budget.md` - piped-mode performance budgets and release thresholds
- `docs/review-prompt.md` - final gate review prompt for refactor/feature planning

## Legacy docs (deprecated)

- `docs/implementation-handbook.md`

If legacy docs conflict with code or with `docs/current-implementation.md`, treat legacy content as outdated.
